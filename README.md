# Задачник NSL · Лига нутрициологии

Командный трекер задач для компании **Лига нутрициологии (NSL)**. Позволяет
ставить задачи разным отделам, назначать ответственных и сроки и отслеживать
выполнение на Kanban-доске.

## Возможности

- **Постановка задач по отделам** — 10 отделов компании (Образование, Методология,
  Кураторы, Маркетинг, Продажи, Контент и SMM, Поддержка, Продукт и IT, HR, Финансы).
- **Ответственный и сроки** — назначение исполнителя из списка сотрудников и дедлайна;
  просроченные и «горящие» задачи подсвечиваются.
- **Kanban-доска** с drag & drop между статусами: Бэклог → К выполнению → В работе →
  На проверке → Готово.
- **Список** — компактное табличное представление тех же задач.
- **Приоритеты** — низкий / средний / высокий / срочный.
- **Фильтры и поиск** — по отделу, ответственному, приоритету, тексту и тегам,
  сортировка по приоритету и сроку, фильтр «только просроченные».
- **Обзорная панель** — ключевые метрики (всего / выполнено / просрочено / горящие),
  распределение по отделам и кольцо прогресса.
- **Локальное хранение** — все данные сохраняются в `localStorage` браузера.

## Технологии

React 18 + Vite. Без внешнего бэкенда — данные живут в браузере.

## Запуск

```bash
npm install     # установить зависимости
npm run dev     # запустить дев-сервер (http://localhost:5173)
npm run build   # собрать продакшн-версию в dist/
npm run preview # предпросмотр собранной версии
```

## Структура

```
src/
  data.js               # справочники (отделы, сотрудники, статусы, приоритеты) и демо-задачи
  useStore.js           # хранилище задач + localStorage, фильтрация, работа со сроками
  App.jsx               # корневой компонент, фильтры и переключение представлений
  components/
    Sidebar.jsx         # навигация и отделы
    Board.jsx           # Kanban-доска с drag & drop
    TaskCard.jsx        # карточка задачи + аватары
    TaskList.jsx        # табличное представление
    TaskModal.jsx       # создание и редактирование задачи
    Dashboard.jsx       # обзорная панель со статистикой
```

## Дальнейшее развитие

Приложение готово к подключению бэкенда (например, Node.js + база данных) вместо
`localStorage`: вся работа с данными изолирована в `src/useStore.js`.
