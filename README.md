# Финансовая аналитика из amoCRM · «План-Факт»

Программа с веб-интерфейсом, которая берёт финансовые данные **напрямую из amoCRM**
и показывает отчёт «План-Факт» (как в ваших Google Таблицах), но без ручной выгрузки.

Показатели: лиды (план/факт), квал-лиды, продажи, выручка, деньги в кассу,
рассрочка, предоплата, конверсия, средний чек — по проектам (Фитнес, Нутрициология),
по менеджерам и по источникам заявок.

---

## Быстрый старт (демо-режим, без amoCRM)

Чтобы просто посмотреть, как выглядит программа:

1. Установите Python 3.10+ (если ещё нет): https://www.python.org/downloads/
2. Откройте терминал (командную строку) в папке с программой и выполните:

   ```bash
   pip install -r requirements.txt
   python run.py
   ```

3. Откроется браузер с адресом `http://127.0.0.1:5000` и тестовыми данными.

> В демо-режиме используются выдуманные данные — реальный amoCRM не нужен.

## Подключение к вашему amoCRM

### Шаг 1. Получить доступ к API amoCRM

1. Зайдите в amoCRM → **Настройки → Интеграции → Создать интеграцию**
   (выберите «Внешняя интеграция» / приватную).
2. Дайте интеграции доступ на чтение сделок.
3. На вкладке **«Ключи и доступы»** возьмите **долгосрочный токен**
   (long-lived access token) — длинная строка.
4. Запомните свой **поддомен**: если вы заходите по адресу
   `https://myschool.amocrm.ru`, то поддомен — это `myschool`.

### Шаг 2. Вписать данные в программу

1. Скопируйте файл `.env.example` в `.env`.
2. Откройте `.env` и заполните:

   ```
   AMOCRM_SUBDOMAIN=myschool
   AMOCRM_ACCESS_TOKEN=ваш_длинный_токен
   DEMO_MODE=0
   ```

   `DEMO_MODE=0` переключает программу на реальные данные amoCRM.

3. Запустите заново: `python run.py`.
   Вверху панели должно появиться «amoCRM подключён».

### Шаг 3. Настроить показатели под ваш amoCRM

Файл **`app/mapping.py`** — главная настройка отчёта. В нём указывается:

- **Проекты (воронки).** Для каждого проекта — `pipeline_id` воронки amoCRM.
- **Названия полей сделки** — откуда брать «деньги в кассу», «рассрочка»,
  «предоплата», «квал-лид», «менеджер», «источник».
  Точные названия видны в amoCRM: Настройки → Поля → Сделки.
- **Планы на месяц** — цели по лидам, продажам и выручке для сравнения с фактом.

> Как узнать id воронок и точные названия полей — обратитесь ко мне,
> и я помогу сопоставить их по скриншоту настроек amoCRM.

## Что где лежит

```
run.py                  — запуск программы
requirements.txt        — зависимости Python
.env.example            — шаблон настроек (скопировать в .env)
app/
  server.py             — веб-сервер (страницы и API)
  amocrm_client.py      — подключение к API amoCRM
  analytics.py          — расчёт показателей «План-Факт»
  mapping.py            — НАСТРОЙКА: воронки, поля, планы
  demo_data.py          — тестовые данные для демо-режима
  templates/index.html  — страница дашборда
  static/               — оформление (CSS) и логика (JS)
```

## Частые вопросы

**«amoCRM отклонил токен (401)»** — токен неверный или устарел.
Проверьте `AMOCRM_ACCESS_TOKEN` в `.env`.

**Показатели пустые / нули** — вероятно, не совпадают названия полей
в `app/mapping.py` с реальными полями amoCRM. Сверьте названия.

**Хочу вернуть демо-режим** — поставьте `DEMO_MODE=1` в `.env`.
